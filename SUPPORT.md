# Support

Turkce surum: [SUPPORT.tr.md](./SUPPORT.tr.md)

## Where to ask for help

- Bugs and reproducible product problems: open a GitHub issue
- Feature requests and scoped improvements: open a GitHub issue
- Security-sensitive reports: use the private reporting path in `SECURITY.md` or `.github/SECURITY.md`

## What to include

- Your Node.js and npm versions
- How you launched the server
- Whether `MCP_GITPRO_GITHUB_TOKEN` was set
- The relevant config values from `mcp-gitpro.config.json` with secrets removed
- The command output from `npm run ci:check` if the problem is build or test related

## What this repo does not support

Support requests outside the documented product scope are usually out of scope too, including local git automation, filesystem editing, shell execution, browser automation, and HTTP transport in v1.
